# EveryUtils AI Agent Guide

## Mission

Build the best collection of modern web utilities.

---

## Before writing code

Always read, in this order:

1. .ai/project.md
2. .ai/architecture.md
3. .ai/business-rules.md
4. .ai/context.md
5. Current GitHub Issue

---

## Rules

- Never modify code outside the issue scope.
- Prefer small Pull Requests.
- Always update documentation when behaviour changes.
- Write tests whenever possible.
- Never break existing tools.
- Prefer reusable components.
- If an architectural decision changes, update `.ai/decisions.md`.

---

## At the end of every task

Update:

- context.md
- changelog
- documentation

before opening the Pull Request.
